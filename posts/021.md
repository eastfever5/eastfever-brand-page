# Antigravity AI 모델별 특징 파헤치기

최근 구글에서 출시한 AI 통합 개발 환경인 **Antigravity**를 사용하면서, 자연스레 안티에서 제공하는 AI 모델들의 각 특성에 대해 깊이 있는 고민이 생겼습니다. 

"대충 Pro가 Fast보다 느리지만 더 똑똑하겠지", "클로드가 문장력이 좋으니 글쓰기나 전략 세우기에 써야지" 수준으로 적당히 골라가며 쓰고 있었지만, 한 달 사용량이 금방 소진되는 것을 보며 각 모델의 장단점을 정확히 파악해 효율적으로 사용해야겠다는 생각이 들었습니다.

현재 Antigravity에서 사용할 수 있는 6가지 모델을 분석하고 정리한 내용을 공유합니다.

---

## 1. Gemini Pro (High / Low)

구글의 기술력이 집약된 핵심 모델들입니다.

- **Gemini Pro (High)**: 
  - **특징**: 가장 깊이 생각하고 정교한 추론을 수행하는 모델입니다.
  - **활용**: 새로운 프로젝트의 전체적인 구조를 잡거나, 복잡하고 중요한 로직의 리팩토링, 고난이도 문제 해결에 사용합니다.
  - **주의**: 처리 속도가 느리고 할당량 소모가 크기 때문에 신중하게 사용해야 합니다.

- **Gemini Pro (Low)**: 
  - **특징**: 속도와 성능의 균형을 맞춘 밸런스 타입입니다.
  - **활용**: 단순한 질문보다는 복잡하지만 High 모델까지는 필요 없는 애매한 작업들에 적합합니다.

## 2. Gemini Flash

압도적인 퍼포먼스를 자랑하는 '가성비' 모델입니다.

- **특징**: 응답 속도가 매우 빠르며, 단순 질문이나 간단한 UI 수정 작업에 최적화되어 있습니다.
- **활용**: 최근에는 대부분의 가벼운 구현 작업을 Flash 위주로 진행하고 있습니다. 기본적인 신규 기능 구현도 충분히 잘 해내기 때문입니다.
- **팁**: Flash로 해결이 어려운 문제에 부딪혔을 때만 Pro 모델의 힘을 빌리는 방식으로 할당량을 관리하는 것이 좋습니다.

## 3. Claude (Opus / Sonnet)

Anthropic의 모델들로, 특정 영역에서 독보적인 강점을 보입니다.

- **Claude Opus**: 
  - **특징**: 코딩 벤치마크 성능이 업계 최상급으로 평가받습니다.
  - **활용**: 수십 개에서 수백 개의 파일로 구성된 복잡한 프로젝트 구조를 분석하거나, 정교한 백엔드 로직을 설계할 때 최고의 성능을 발휘합니다.

- **Claude Sonnet**: 
  - **특징**: 사용자의 의도를 센스 있게 파악하는 능력이 탁월합니다.
  - **활용**: 프론트엔드 레이아웃 구성이나 CSS 작업 시 시각적 감각이 훌륭하며, 문서 작문 및 설계 초안 작성에도 뛰어납니다.

## 4. GPT-OSS

OpenAI 기반이지만 오픈소스 프로젝트에 특화되어 튜닝된 모델입니다.

- **특징**: 잘 알려진 라이브러리나 프레임워크에 대한 이해도가 매우 높습니다.
- **활용**: 특히 '코드 리뷰' 모드에서 다른 모델들이 놓치기 쉬운 세밀한 버그를 잡아내는 데 뛰어난 성능을 보인다는 평이 많습니다. 
- **참고**: 안티그라비티 내에서는 클로드와 사용량 할당이 묶여 있는 경우가 많아, 상황에 따른 전략적 선택이 필요합니다.

## 요약 및 추천 활용법

프로젝트 진행 시 다음과 같은 기준으로 모델을 선택해 보시기 바랍니다.

| 작업 유형 | 추천 모델 |
| :--- | :--- |
| **기본 기획 및 일반 작업** | Gemini Flash / Low |
| **심층적 작문 및 초기 설계** | Claude Sonnet |
| **복잡한 구조 분석 및 고급 코딩** | Gemini Pro High / Claude Opus |
| **세밀한 코드 리뷰 및 버그 수정** | GPT-OSS |

AI 개발 도구의 할당량은 한정되어 있습니다. 각 모델의 '성격'을 잘 파악하여 적재적소에 배치한다면 훨씬 더 쾌적하고 효율적인 개발 여정이 될 것입니다. 

오늘 정리한 내용이 여러분의 **앤트로픽(Antigravity)** 활용에 작은 가이드가 되길 바랍니다.
