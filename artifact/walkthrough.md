# 워크스루: About 페이지 이슈 해결 및 캐시 버스팅 적용

## 1. 개요
About 페이지(`about/index.html`)의 마크다운 렌더링 문제를 해결하고, 수정 사항이 브라우저에 즉시 반영되도록 캐시 버스팅(Version Busting)을 적용했습니다.

## 2. 주요 변경 사항

### [JS] 데이터 로딩 안정화 및 캐시 방지 (`js/data-loader.js`)
- `fetch` 시 절대 경로(`window.location.origin`)를 우선 사용하고, 실패 시 상대 경로로 시도하는 폴백 로직을 적용했습니다.
- `data.json` 요청 시 타임스탬프를 추가하여 데이터 캐시 문제를 해결했습니다.

### [HTML] 마크다운 로딩 및 캐시 버스팅 (`about/index.html`, `post.html`)
- **컨텐츠 캐시 방지**: `about.md` 및 개별 포스트(`posts/*.md`)를 페치할 때 `?v=[timestamp]` 파라미터를 추가하여 수정 즉시 반영되도록 했습니다.
- **정적 자산 버전업**: 모든 주요 HTML 파일의 CSS 및 JS 링크에 `?v=5` 파라미터를 적용했습니다.
- **렌더링 로직 개선**: i18n 초기화와 마크다운 로딩을 분리하여 안정성을 높이고, 정규식을 통한 Frontmatter 제거를 구현했습니다.

## 3. 검증 결과

### 수정 사항 반영 확인
- 사용자가 `about.md`에서 삭제한 '주요 서비스' 섹션이 브라우저 새로고침 시 즉시 사라지는 것을 확인했습니다.
- 데스크탑 환경에서 마크다운 헤더와 리스트 스타일이 정상적으로 적용됩니다.
